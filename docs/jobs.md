# Jobs

## What a job is

A job represents work performed for a customer (service call, repair, install, etc.).

## Common workflows

- Create job for a customer
- Track job status
- Prioritize work
- Assign jobs to users (future enhancement if not surfaced in UI yet)

## Fields

- Customer (required)
- Title (required)
- Description (optional)
- Status (defaults to `open`)
- Priority (defaults to `medium`)
